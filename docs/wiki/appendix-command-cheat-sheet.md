# Appendix C: Command Cheat Sheet

This appendix is the short "I do not want to hunt around for the right script
name" page.

## Everyday commands

### Open the full main workflow

```powershell
.\scripts\launch-visualizer-workbench.ps1
```

### Open the browser dashboard

```powershell
.\scripts\launch-browser-control-center.ps1
```

### Open the native operator app

```powershell
.\scripts\launch-visualizer-studio.ps1
```

### Open the live log viewer

```powershell
.\scripts\launch-activity-monitor.ps1
```

### Run the full quality suite

```powershell
.\scripts\run-all-quality-checks.ps1
```

### Build the beginner wiki

```powershell
.\scripts\build-beginner-wiki.ps1
```

### Serve the beginner wiki locally

```powershell
.\scripts\serve-beginner-wiki.ps1
```

## A good memory trick

If the command starts with `launch-`, it usually opens a tool.

If it starts with `test-`, `lint-`, `typecheck-`, or `measure-`, it usually
checks part of the project.
