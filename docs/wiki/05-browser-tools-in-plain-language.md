# Chapter 5: Browser Tools in Plain Language

The browser tools can feel abstract until you use them with a simple question
in mind.

So let us give each browser tool one plain-language purpose.

## Control Center

The Control Center answers questions like:

- Is this machine ready?
- Can I build right now?
- Can I run builds and tests?
- What docs should I read?
- What does the API return?
- Is the managed app running?

Use it when your brain is in:

- setup mode
- diagnostics mode
- testing mode
- API-inspection mode

## Activity Monitor

The Activity Monitor answers:

"What are all the running pieces doing over time?"

It gives you:

- one shared timeline
- sorting
- filtering
- cross-app visibility

## Scene Studio

Scene Studio answers:

"Can I stay in the browser and still experiment with scene generation?"

Use it when you want:

- a lighter-weight creative surface
- preset-driven scene experiments
- browser-side interaction without living in the desktop app

## How the browser tools relate to each other

They are not rivals.

They are different angles on the same system:

- Control Center is operational
- Activity Monitor is observational
- Scene Studio is creative

## A practical workflow

One beginner-friendly workflow looks like this:

1. Open Control Center.
2. Check prerequisites and health.
3. Open Activity Monitor in another tab.
4. Use Scene Studio or Visualizer Studio to generate a scene.
5. Watch the logs update while you do it.
