What this app is for

Signal Router is the desktop-side data router for Halcyn. Instead of asking one tool to do everything, the responsibilities are now cleaner:

  • The desktop render control panel focuses on scene presets and visual tuning.
  • Bars Studio focuses on bar-wall-specific rendering options.
  • Signal Router focuses on capturing or generating live data and sending it to one or both renderer modes.
Beginner mental model: this app is the data source brain. The other desktop apps are the scene editors.

Beginner walkthrough

  1. Start the scene renderer with .\scripts\launch-visualizer.ps1 if you want the regular scene presets.
  2. Start the bar-wall renderer with .\scripts\launch-visualizer.ps1 -Sample spectrograph -Port 8090 if you want the bar-wall mode.
  3. Run .\scripts\launch-signal-router.ps1.
  4. Pick one source mode such as JSON document, Plain text, Random values, Audio device, Pointer pad, or External feed.
  5. Enable the target or targets you want to drive.
  6. Click Preview to see whether the data source produced sensible values.
  7. Click Validate to ask the renderer APIs whether they accept the generated scenes.
  8. Click Apply or Start live stream when the preview looks right.

Source modes

JSON document

Parse arbitrary nested JSON, flatten numbers directly, turn booleans into 1.0 or 0.0, and convert strings into UTF-8 bytes.

Plain text

Turn one plain-text string into byte values without requiring a JSON wrapper first.

Random values

Generate a repeatable pseudo-random value stream from a seed so testing is easier.

Audio device

Capture either output loopback or input-device sound, then route that live audio analysis into both renderer families.

Pointer pad

Use local pointer X, Y, and motion speed when you want a simple interactive source with no hardware dependencies.

External feed

Follow the newest JSON document delivered by another local helper app such as Audio Sender.

How the same source reaches two different renderers

  1. The source mode produces one numeric stream.
  2. The shared source builder summarizes that numeric stream into smaller signal values for the scene renderer.
  3. The same numeric stream is also packaged as JSON for the bar-wall renderer.
  4. The scene renderer receives a normal Halcyn scene generated through the existing desktop scene builder.
  5. The bar-wall renderer receives a normal Halcyn scene generated through the Bars Studio scene builder.

The important design choice is that both outputs are still ordinary Halcyn scene JSON. The new work lives in the translation step, not in inventing a second network protocol.

When you want to inspect what happened across Audio Sender, Signal Router, and Bars Studio, open .\scripts\launch-activity-monitor.ps1. The shared activity monitor reads structured desktop logs from all of those tools in one place.
