What the Control Center does

The Control Center is a separate Python server that can run before the C++ application exists. That means it can help with first-time setup, build orchestration, test execution, docs access, API experiments, and log viewing from one browser tab.

This documentation page is static study material. The live Control Center itself runs locally on your machine after you launch it with the script below. 
.\scripts\launch-browser-control-center.ps1

Main areas in the UI

Dashboard

Shows tool availability, app state, recent jobs, and Control Center logs.

Build & Run

Starts bootstrap, build, format, docs generation, and the managed app process.

Testing

Runs smoke checks against the live API and summarizes the results.

API Lab

Sends browser-originated requests through the Control Center into the live Halcyn API.

Scene Studio

Opens the separate preset-driven scene authoring surface for live visual manipulation.

Why this server is separate from the C++ app

A build-orchestration dashboard is most useful before the C++ app has been built. Keeping it separate lets it bootstrap the rest of the project instead of depending on it.
