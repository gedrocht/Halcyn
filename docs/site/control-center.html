What the control plane does

The control plane is a separate Python server that can run before the C++ application exists. That means it can help with first-time setup, build orchestration, test execution, docs access, API experiments, and log viewing from one browser tab. 

.\scripts\studio.ps1

The button above points at the control-plane root path. That works when these docs are being served by the control plane itself. If you are viewing the docs from the standalone docs server, launch .\scripts\studio.ps1 separately and open the address it prints.

Main areas in the UI

Dashboard

Shows tool availability, app state, recent jobs, and control-plane logs.

Build & Run

Starts bootstrap, build, format, docs generation, and the managed app process.

Testing

Runs smoke checks against the live API and summarizes the results.

API Lab

Sends browser-originated requests through the control plane into the live Halcyn API.

Client Studio

Opens the separate preset-driven scene GUI for live visual manipulation.

Why this server is separate from the C++ app

A build-orchestration dashboard is most useful before the C++ app has been built. Keeping it separate lets it bootstrap the rest of the project instead of depending on it.
