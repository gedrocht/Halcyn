What the Control Center does

The Control Center is a separate Python web server that can run even before the native Visualizer has been built. That separation is intentional: it lets the browser dashboard bootstrap the project, launch the renderer, inspect docs, and probe the API without depending on the renderer already existing. 

.\scripts\launch-browser-control-center.ps1

Activity Monitor

Opens the shared, sortable, filterable log viewer for all participating apps.

Why the Activity Monitor lives here

The shared activity journal is meant to be viewed in a browser because that gives you easy sorting, filtering, and cross-app visibility. The Control Center already owns the local browser-facing server, so it is the natural place to host the Activity Monitor route. 

.\scripts\launch-activity-monitor.ps1

How it fits with the rest of the repo

The Control Center is not the renderer and it is not the native operator app. It is the orchestration layer around them. The renderer draws. Visualizer Studio generates and sends scenes. The Control Center helps you launch, inspect, and test both.
