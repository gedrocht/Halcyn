What it is

Client Studio is the separate browser-facing scene authoring surface for Halcyn. It lives in its own top-level folder, but it still relies on the existing control-plane server for routing, live-session control, logging, and test coverage. 

http://127.0.0.1:9001/client/

Signal inputs

Unix time

Provides a stable animated phase even when no hardware input is active.

Noise

Adds deterministic variation so repeated frames do not feel too rigid.

Pointer

Uses normalized mouse position and motion speed captured in the browser.

Microphone

Uses browser-side microphone analysis when the user explicitly enables it.

Manual drive

Lets the operator push the preset without depending on external input at all.

Why this stays in the same repo

The client and renderer share one scene contract. Keeping the client in this repository means the contract, docs, CI, and release rules stay synchronized. The UI is still physically separated under client_studio/, so it can evolve independently without dragging the rest of the control plane around.

How live streaming works now

The browser does not need to send a full JSON scene every time the pointer or audio input changes. Instead, it sends lighter control updates to the control plane. The control plane keeps a persistent live session, regenerates scenes server-side on a fixed cadence, and streams those scenes into the live Halcyn API.

Why it is still build-free HTML instead of React

The main latency improvement came from moving the streaming loop to the server side, not from changing frontend frameworks. Keeping the client as a static HTML, CSS, and JavaScript app preserves the repo's existing quality gates on machines without a Node toolchain. If the UI grows more state-heavy later, the backend contract is now stable enough to support a React migration without changing the renderer-facing design.

Control-plane routes behind the client

Client Studio uses the catalog, preview, one-shot apply, and live-session routes under /api/client-studio/.... The live-session endpoints are what make the browser feel more responsive during continuous manipulation.
