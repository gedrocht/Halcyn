What it is

Client Studio is the separate browser-facing scene authoring surface for Halcyn. It lives in its own top-level folder, but it still relies on the existing control-plane server for routing, validation, logging, and test coverage. 

http://127.0.0.1:9001/client/

Signal inputs

Unix time

Provides a stable animated phase even when no hardware input is active.

Noise

Adds deterministic variation so repeated frames do not feel too rigid.

Pointer

Uses normalized mouse position and motion speed captured in the browser.

Microphone

Uses browser-side microphone analysis when the user explicitly enables it.

Manual drive

Lets the operator push the preset without depending on external input at all.

Why this stays in the same repo

The client and renderer share one scene contract. Keeping the client in this repository means the contract, docs, CI, and release rules stay synchronized. The UI is still physically separated under client_studio/, so it can evolve independently without dragging the rest of the control plane around.

Safety path for live apply

The browser does not submit raw ad-hoc values straight to the renderer. It first asks the control-plane translator to generate a scene, then the control plane validates that scene against the live Halcyn API, and only then does it submit the scene for activation.
