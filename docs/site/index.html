Overview

One renderer. One desktop studio. One browser dashboard. One shared activity timeline.

Halcyn is a C++20 OpenGL renderer with an embedded HTTP API. It accepts JSON scene descriptions, validates them, stores the latest valid scene, and renders that scene on the GPU. Around that renderer sits one browser Control Center, one browser Activity Monitor, one browser Scene Studio, and one unified native desktop operator app called Visualizer Studio.

C++20 OpenGL 3.3 2D + 3D scenes Bar-wall scenes Shared activity log

The current supported workflow

Visualizer

The native C++ app that renders either preset scenes or bar-wall scenes.

Visualizer Studio

The unified native desktop operator app for sources, previews, validation, apply, and live streaming.

Control Center

The browser dashboard for setup, build, test, docs, managed-app launching, and API experiments.

Activity Monitor

The sortable, filterable browser log viewer for the shared activity journal written by all running tools.

Scene Studio

The browser-side creative surface for preset-driven scene generation and browser-only live control.

What changed in the unified workflow

Earlier iterations split desktop responsibilities across several specialized apps. That made the code interesting, but it also made the workflow harder to learn. The current repo now treats bar-wall rendering as one more scene family inside the same Visualizer instead of as a separate renderer executable, and it treats Visualizer Studio as the single supported native desktop control surface.

The older split desktop launchers still exist as compatibility wrappers, but the supported beginner story is now: Visualizer + Visualizer Studio + Control Center + Activity Monitor.

Project tour

Path Why it exists
src/scene_description Scene types, validation, JSON codec, and built-in example scenes.
src/shared_runtime Thread-safe scene storage plus the native runtime log that now also feeds the shared activity journal.
src/http_api The embedded HTTP API that exposes health, examples, validation, submission, and runtime logs.
src/opengl_renderer Window creation, shaders, GPU uploads, and the real rendering loop.
src/desktop_app The top-level application glue that wires the Visualizer together.
browser_control_center The browser dashboard and Activity Monitor server.
browser_scene_studio The separate browser-side preset authoring surface.
desktop_visualizer_operator_console The unified native Visualizer Studio package.
desktop_shared_control_support Shared support code for activity logging, audio capture, and renderer API calls.
scripts Build, launch, test, docs, and workbench entry points.

Recommended learning order

  1. Read Tutorial.
  2. Read Architecture.
  3. Run .\scripts\launch-visualizer-workbench.ps1.
  4. Use Visualizer Studio to preview and apply both a preset scene and a bar-wall scene.
  5. Open the Activity Monitor and watch the shared log stream update while you do that.
  6. Use the Control Center API Lab to inspect the same renderer over HTTP.
