What changed

Earlier versions of the project described a separate spectrograph renderer and a separate spectrograph desktop workflow. The current project keeps the visual idea, but it no longer treats it as a separate renderer. Bar-wall rendering is now one scene family inside the same Visualizer app.

The compatibility route and compatibility launcher still exist: GET /api/v1/examples/spectrograph and .\scripts\launch-halcyn-spectrograph-app.ps1. They now forward to the same unified renderer behavior.

How generic data becomes a bar wall

  1. Collect one live source snapshot from JSON, text, random values, audio, or pointer motion.
  2. Flatten that source into one numeric stream.
  3. Maintain a rolling history of past values.
  4. Estimate a useful active range from the rolling history, or use a manually supplied range.
  5. Split the current value stream into equal-sized groups for an N x N wall.
  6. Normalize each group against the chosen active range.
  7. Build one 3D bar per group and color it according to the selected presentation style.
  8. Submit the resulting ordinary Halcyn 3D scene to the unified Visualizer.

Where to control it now

Use .\scripts\launch-visualizer-studio.ps1. Inside Visualizer Studio:

  • choose Bar wall as the scene family
  • choose a live source mode
  • adjust grid size, shader style, anti-aliasing, and range mode
  • preview, validate, apply, or live-stream the scene

If you want the renderer to begin on the bar-wall family immediately, use: 

.\scripts\launch-halcyn-spectrograph-app.ps1

Preferred example route

The preferred example route is now: 

GET /api/v1/examples/bar-wall

The old spectrograph example route remains as a compatibility alias: 

GET /api/v1/examples/spectrograph

Render-style controls

Bar-wall scenes are still ordinary 3D scenes, but they often include the optional renderStyle object: 

{
  "renderStyle": {
    "shader": "heatmap",
    "antiAliasing": true
  }
}

Visualizer Studio exposes those same controls directly in its desktop UI, so you do not need to hand-edit JSON if you are just learning.

How to study this feature

  1. Launch .\scripts\launch-visualizer-workbench.ps1.
  2. In Visualizer Studio, switch the scene family to Bar wall.
  3. Try a plain-text source first so you can see how strings become UTF-8 bytes and then bar heights.
  4. Then try audio input or output capture and watch the bar wall respond live.
  5. Open the preview JSON window and compare the source data to the generated 3D scene payload.
