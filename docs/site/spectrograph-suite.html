What the bar-wall workflow is

The bar-wall workflow is a coordinated tool set:

  • .\scripts\launch-visualizer.ps1 -Sample spectrograph -Port 8090 starts the renderer in its bar-wall personality.
  • .\scripts\launch-signal-router.ps1 opens the central live-source router that can drive the scene renderer, the bar-wall renderer, or both.
  • .\scripts\launch-bars-studio.ps1 opens the specialized deep-tuning console for bar-wall-specific behavior.
  • .\scripts\launch-audio-sender.ps1 opens a companion helper app that captures live audio and feeds it into Signal Router or Bars Studio.
The renderer still consumes ordinary Halcyn scene JSON. The new work lives in how the desktop tools interpret arbitrary source data and shape it into a bar-wall scene.

If you want the quickest beginner path, run .\scripts\launch-bars-workbench.ps1. That one helper opens the Visualizer, Signal Router, and Audio Sender in separate windows so you can see the whole workflow without memorizing several commands first.

How generic JSON becomes bars

  1. Parse the raw JSON text.
  2. Flatten nested arrays, objects, booleans, and numbers into one numeric stream.
  3. Convert strings into UTF-8 byte values so even text can still move the bars.
  4. Append the new values into a rolling history buffer.
  5. Compute observed minimum, observed maximum, mean, and standard deviation.
  6. Choose either an automatic range based on those statistics or a manual minimum/maximum range set by the operator.
  7. Group the current source values into exactly N x N equally sized buckets.
  8. Normalize each bucket into a 0..1 intensity.
  9. Build one colored 3D bar prism per bucket and submit the resulting Halcyn scene JSON.

What the desktop control panel can change

Data source

Paste generic JSON directly, load a JSON file, pick a built-in example, generate a fresh random numeric sample, or route live helper-fed JSON.

Range model

Switch between automatic statistical normalization and manually supplied minimum and maximum values.

Bar wall size

Choose the N in the final N x N bar grid.

Render presentation

Toggle multisample anti-aliasing and choose between standard, neon, and heatmap shader styles.

Geometry shape

Change floor height and peak height so the bars can feel flatter, denser, taller, or more dramatic.

External helper feed

Let the router or Bars Studio follow live JSON arriving from Audio Sender while preserving the same preview, validate, apply, and live-stream workflow.

Live operation

Preview, validate, apply once, or continuously live-stream at a chosen cadence.

If you want one shared live source to drive both the scene renderer and the bar-wall renderer, use the Signal Router.

If you want a helper app focused only on live audio capture for this renderer family, study the Audio Sender.

Useful API and scene details

The bar-wall mode exposes the same core API routes as the main app and adds a built-in example route: 

GET /api/v1/examples/spectrograph

Spectrograph scenes are still normal 3D scenes, but they often make use of the optional renderStyle object: 

{
  "renderStyle": {
    "shader": "heatmap",
    "antiAliasing": true
  }
}

Recommended learning order

  1. Either launch the workflow manually or start with .\scripts\launch-bars-workbench.ps1.
  2. Confirm the Visualizer window appears and shows its startup bar wall.
  3. Open Signal Router and load one built-in example or switch to External feed.
  4. Then use Audio Sender, start output or input capture, and watch the bridge feed live JSON into Signal Router.
  5. Switch between automatic and manual range modes and watch how the bars respond.
  6. Change the bar-grid size to see how grouping changes the final scene.
  7. Open the generated scene JSON window and compare the source data to the final renderer payload.
  8. Then open Bars Studio if you want more specialized bar-wall-only tuning.
