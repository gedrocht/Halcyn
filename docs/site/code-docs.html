What is already documented in the source?

Public classes, structs, and functions have descriptive comments in the headers. These comments explain purpose and usage so that a reader can often learn a class from its interface before reading the implementation.

How to generate full HTML code docs

.\scripts\generate-code-reference-docs.ps1

The generated site is written to docs/generated/code-reference. The included Doxyfile already scans src and README.md.

Recommended reading order

  1. Read src/scene_description/SceneTypes.hpp to understand the data model.
  2. Read src/scene_description/SceneJsonCodec.hpp and .cpp to learn how JSON becomes a scene.
  3. Read src/shared_runtime/SceneStore.hpp to see how the latest scene is shared safely.
  4. Read src/http_api/ApiServer.hpp to learn the HTTP surface.
  5. Read src/opengl_renderer/Renderer.hpp and .cpp last, because it is the most graphics-heavy layer.
