Why the tests are layered instead of all living in one place

Halcyn now has native C++ rendering code, browser-side Python tooling, and a desktop family of Python control panels. The deterministic pieces are pushed into automated tests, while the genuinely hardware-dependent pieces still keep a smaller manual-check list.

Current automated coverage

Native scene pipeline

Valid 2D and 3D payloads are parsed, validated, stored, and normalized predictably.

Browser Control Center

Lint, type checks, coverage, and repository-contract tests keep the orchestration UI honest.

Desktop control panel

Unit tests cover audio service fallbacks, HTTP client behavior, controller logic, and the hidden Tk window flow.

Bars Studio

Unit tests cover generic JSON flattening, rolling-range normalization, controller live streaming, and the Tk-based bar-wall operator window.

Audio Sender

Unit tests cover device-flow selection, bridge delivery, rolling audio-history packaging, controller live sending, and the Tk-based helper window.

Signal Router

Unit tests cover source-mode routing, dual-target preview/apply behavior, settings documents, audio capture integration, detached preview windows, and the shared bridge wiring.

Recommended manual checks

  • Post the 2D triangle sample and confirm the window updates.
  • Post the 3D tetrahedron sample and confirm depth ordering looks correct.
  • Post intentionally invalid JSON and confirm the API returns a 400 response with readable errors.
  • Resize the window and confirm the scene stays visible.
  • Launch the desktop render control panel and confirm it can preview, validate, and apply a scene.
  • If you have audio hardware available, confirm the desktop panel can enumerate a device and react to changing input levels.
  • Launch the Visualizer in bar-wall mode and Bars Studio, load a string-heavy example, and confirm it still produces a valid moving bar wall.
  • Launch Audio Sender, choose an active output or input source, and confirm the volume monitor responds before sending data to a local bridge.
  • Launch Signal Router, enable both targets, and confirm one source stream can validate and apply scenes into both Visualizer modes.
