Why the tests focus on the domain and core layers

The domain layer and the scene store are deterministic. Given the same JSON input, they should always produce the same result. That makes them excellent unit test candidates. A real render window depends on GPU drivers and hardware availability, so that part is better handled with manual checks.

Scene JSON parsing

Valid 2D and 3D payloads are accepted and turned into typed scenes.

Validation rules

Broken cameras or malformed primitive counts produce readable errors.

Scene store behavior

Version numbers increase and scene replacement behaves predictably.

How to run the tests

.\scripts\test.ps1
.\scripts\test.ps1 -Configuration Release

Recommended manual checks

  • Post the 2D triangle sample and confirm the window updates.
  • Post the 3D tetrahedron sample and confirm depth ordering looks correct.
  • Post intentionally invalid JSON and confirm the API returns a 400 response with readable errors.
  • Resize the window and confirm the scene stays visible.
