Base idea

The Visualizer has one API. It does not have a separate "spectrograph API" anymore. Bar-wall rendering is just one kind of 3D scene the same renderer can display.

Core endpoints

GET  /api/v1/health
GET  /api/v1/scene
GET  /api/v1/runtime/limits
GET  /api/v1/runtime/logs?limit=200
GET  /api/v1/examples/2d
GET  /api/v1/examples/3d
GET  /api/v1/examples/bar-wall
GET  /api/v1/examples/spectrograph
POST /api/v1/scene/validate
POST /api/v1/scene

/api/v1/examples/spectrograph is a compatibility alias for the same built-in bar-wall sample returned by /api/v1/examples/bar-wall.

What usually calls this API

Scene validation flow

  1. Send a candidate scene to POST /api/v1/scene/validate.
  2. If the response is valid, submit the same scene to POST /api/v1/scene.
  3. The scene store updates, and the renderer uses the new scene on the next frame.
