Endpoints

Method Path Purpose
GET/api/v1/healthService status plus current scene summary.
GET/api/v1/sceneThe current active scene as JSON.
GET/api/v1/runtime/limitsThe current request, vertex, and index safety limits.
GET/api/v1/runtime/logs?limit=200The newest in-memory runtime log entries for diagnostics.
GET/api/v1/examples/2dA built-in 2D example scene.
GET/api/v1/examples/3dA built-in 3D example scene.
GET/api/v1/examples/spectrographA built-in 3D bar-grid spectrograph example scene.
POST/api/v1/scene/validateValidate a scene without activating it.
POST/api/v1/sceneValidate and activate a new scene.

2D scene payload

Set sceneType to "2d". Every 2D vertex must include x, y, r, g, b, and a.

{
  "sceneType": "2d",
  "primitive": "triangles",
  "vertices": [
    { "x": -1.0, "y": -1.0, "r": 1.0, "g": 0.0, "b": 0.0, "a": 1.0 },
    { "x": 0.0, "y": 1.0, "r": 0.0, "g": 1.0, "b": 0.0, "a": 1.0 },
    { "x": 1.0, "y": -1.0, "r": 0.0, "g": 0.0, "b": 1.0, "a": 1.0 }
  ]
}

3D scene payload

Set sceneType to "3d". Every 3D vertex must include x, y, z, r, g, and b. a is optional. A 3D scene may also include renderStyle when the caller wants to choose shader presentation and multisample anti-aliasing.

{
  "sceneType": "3d",
  "primitive": "triangles",
  "camera": {
    "position": { "x": 2.5, "y": 2.0, "z": 2.8 },
    "target": { "x": 0.0, "y": 0.0, "z": 0.0 },
    "up": { "x": 0.0, "y": 1.0, "z": 0.0 },
    "fovYDegrees": 60.0,
    "nearPlane": 0.1,
    "farPlane": 100.0
  },
  "renderStyle": {
    "shader": "heatmap",
    "antiAliasing": true
  },
  "vertices": [
    { "x": -0.8, "y": -0.8, "z": 0.0, "r": 1.0, "g": 0.2, "b": 0.2 },
    { "x": 0.8, "y": -0.8, "z": 0.0, "r": 0.2, "g": 1.0, "b": 0.2 },
    { "x": 0.0, "y": 0.8, "z": 0.0, "r": 0.2, "g": 0.4, "b": 1.0 }
  ],
  "indices": [0, 1, 2]
}

Bar-wall example and presentation fields

The Visualizer uses one shared scene API for both its ordinary scene mode and its bar-wall mode. The difference is on the operator side: tools such as Signal Router and Bars Studio generate a 3D bar-grid scene and often set renderStyle.shader to heatmap or neon. 

GET /api/v1/examples/spectrograph

Validation rules

  • Lines need an even count of vertices or indices.
  • Triangles need counts divisible by three.
  • Scene payloads, vertex counts, and index counts are capped to prevent runaway submissions.
  • fovYDegrees must be between 1 and 179.
  • nearPlane must be positive.
  • farPlane must be greater than nearPlane.
  • camera.target must not equal camera.position.
  • camera.up must not be the zero vector or parallel to the view direction.
  • Every index must point at a real vertex.
  • If you only want to check a payload without changing the active scene, use POST /api/v1/scene/validate.

Browser-based testing and API experiments

The Control Center includes an API Lab for manual requests and a Testing tab for smoke checks. That gives beginners a browser-first way to probe the live API before they are comfortable with command-line tools. 

.\scripts\launch-browser-control-center.ps1

Desktop operator usage

The native desktop tools all talk to the same health, validation, and apply endpoints. Their difference is not a separate API contract, but a different operator experience layered on top of the same HTTP routes. 

.\scripts\launch-desktop-render-control-panel.ps1
.\scripts\launch-signal-router.ps1
.\scripts\launch-bars-studio.ps1

Example PowerShell request

$body = Get-Content -Raw .\examples\scene_2d_triangle.json
Invoke-RestMethod `
  -Method Post `
  -Uri http://127.0.0.1:8080/api/v1/scene `
  -ContentType 'application/json' `
  -Body $body
