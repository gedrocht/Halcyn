What Visualizer Studio is

Visualizer Studio is the one supported native desktop control surface for Halcyn. It replaces the earlier split desktop control panels and gives you one place to choose live sources, choose a scene family, preview JSON, validate the generated scene, apply it once, or live-stream it into the running Visualizer. 

.\scripts\launch-visualizer-studio.ps1
This page is static study material on GitHub Pages. The real app runs locally on your machine because it needs direct access to your desktop UI and optional audio devices.

What it can control

Sources

Choose JSON, plain text, random values, audio devices, or a live pointer pad.

Scene families

Switch between the classic preset scene family and the data-driven bar-wall scene family.

Transport

Preview, validate, apply once, or continuously live-stream to the Visualizer API.

Settings

Revert to defaults, save operator settings, and load them back later.

JSON inspection

Open the current generated scene JSON in a separate preview window and copy it to the clipboard.

Audio capture

Choose output or input sources, start or stop capture, and watch one live volume meter update.

How the two scene families differ

Preset scene family

This is the classic Halcyn workflow. The source data is summarized into smaller control signals and then fed into a chosen preset scene.

Bar-wall scene family

This is the old spectrograph-style workflow, now treated as just another scene family inside the same Visualizer. Generic JSON is flattened into numeric values, normalized against a rolling range, grouped into buckets, and turned into 3D bars.

Why the desktop app is unified now

The older workflow asked you to decide among several desktop tools before you even knew what kind of work you wanted to do. Visualizer Studio simplifies that:

  • one place to choose the source
  • one place to choose the scene family
  • one place to control cadence, validation, apply, and streaming
  • one place to inspect the generated scene JSON

Some older split desktop packages still exist as internal support code, but this unified app is now the one supported public desktop entry point.

Shared logging and diagnostics

Visualizer Studio writes structured events into the shared activity journal. The browser Activity Monitor reads that journal and lets you sort and filter events from:

  • the native Visualizer
  • the browser Control Center
  • Visualizer Studio itself

That means the desktop app no longer has to be its own log island. It participates in one cross-app timeline instead.

Optional Python dependencies

The unified desktop app uses ttkbootstrap for its modern themed UI. Real audio capture uses sounddevice for microphone-style input capture and soundcard for output-loopback capture. 

python -m pip install ttkbootstrap sounddevice soundcard
