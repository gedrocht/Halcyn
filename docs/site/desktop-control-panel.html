What it is

The Desktop Render Control Panel is a native Tkinter application that talks to the same Halcyn HTTP API as the browser tools, but presents the controls in a desktop window that can live beside the renderer itself. 

.\scripts\launch-desktop-render-control-panel.ps1
GitHub Pages can document this tool, but the real control panel runs locally on your machine because it needs direct access to your desktop environment and optional audio hardware.

What it can do

Instant 2D and 3D switching

One native panel can move between desktop-only 2D presets and the shared 3D preset catalog immediately.

Local audio-device selection

Unlike a browser-only surface, the desktop panel can enumerate real local input devices through the optional audio backend.

Preview, validate, and apply

You can inspect the generated JSON, validate it against the live API, or apply it immediately without leaving the window.

Continuous live streaming

The panel can keep submitting scenes on a controlled cadence while you adjust colors, density, motion, and signals.

Main areas in the window

Renderer connection

Choose the API host, port, and live cadence, then run health, validate, apply, start, or stop actions.

Scene controls

Choose scene type, preset, geometry density, motion speed, gain, manual drive, and the active signal families.

Audio input

Refresh local devices, start or stop capture, and route the analyzed level, bass, mid, and treble bands into the current preset.

Pointer pad and preview

Drive pointer-based motion locally and inspect the exact JSON scene the renderer will receive.

How it fits with the browser tools

The browser Control Center is still the orchestration dashboard for builds, tests, logs, docs, and API experiments. The browser Scene Studio is still a browser-native creative surface for live authoring. The Desktop Render Control Panel is the native operator console for the situations where local hardware awareness and a desktop companion window matter more than staying in a browser tab.

Testing and quality gates

The desktop panel is treated like the rest of the repo's maintained code, not like an experimental extra. It has linting, type checks, unit tests, and a 90% coverage gate. 

.\scripts\lint-desktop-render-control-panel.ps1
.\scripts\typecheck-desktop-render-control-panel.ps1
.\scripts\test-desktop-render-control-panel.ps1
.\scripts\measure-desktop-render-control-panel-coverage.ps1

Optional audio dependency

Real device capture depends on the optional sounddevice package. If it is missing, the desktop panel still opens and the rest of the controls still work; only live device capture is disabled. 

python -m pip install sounddevice
