2D vertex fields

FieldMeaning
xThe horizontal position of the vertex in 2D space.
yThe vertical position of the vertex in 2D space.
rThe red color channel for that vertex, usually from 0.0 to 1.0.
gThe green color channel for that vertex, usually from 0.0 to 1.0.
bThe blue color channel for that vertex, usually from 0.0 to 1.0.
aThe alpha channel for that vertex, where 1.0 is fully opaque.

3D vertex and camera fields

FieldMeaning
x, y, zThe 3D position of the vertex.
r, g, bThe vertex color channels used by the shader.
camera.positionWhere the camera is located in world space.
camera.targetThe point the camera looks at.
camera.upThe direction treated as "up" for the camera.
fovYDegreesThe vertical field of view for the 3D projection.
nearPlaneThe near clipping plane distance.
farPlaneThe far clipping plane distance.
renderStyle.shaderThe fragment-shader presentation style for 3D scenes: standard, neon, or heatmap.
renderStyle.antiAliasingWhether OpenGL multisample anti-aliasing should stay enabled for the scene.

Runtime limit fields

The code now enforces maximum payload sizes, vertex counts, and index counts so mistakes become readable validation errors rather than mysterious crashes or frozen windows.

LimitPurpose
maxPayloadBytesCaps how much JSON one request can send.
maxVerticesCaps scene complexity at a beginner-friendly level.
maxIndicesCaps indexed mesh size for 3D submissions.

Desktop spectrograph request fields

FieldMeaning
data.jsonTextThe raw generic JSON text that will be flattened into numeric values.
render.barGridSizeThe N in the final N x N bar wall.
render.shaderStyleThe chosen shader style for the generated 3D bar scene.
render.antiAliasingWhether multisample anti-aliasing should stay enabled.
range.modeWhether the normalization range is automatic or manually specified.
range.manualMinimum / manualMaximumThe operator-defined normalization range when manual mode is selected.
range.rollingHistoryValueCountHow many recent source values should contribute to the rolling statistics.
range.standardDeviationMultiplierHow wide the automatic range should be relative to the rolling standard deviation.
session.cadenceMsThe live-stream interval used by the spectrograph control panel.

Desktop control panel request fields

FieldMeaning
presetIdThe chosen 2D or 3D preset identifier.
target.host / target.portThe Halcyn API endpoint the desktop panel talks to.
settingsEditable visual parameters such as density, size, speed, gain, and colors.
signals.useEpoch, useNoise, usePointer, useAudioToggles that decide which signal families influence the generated scene.
signals.pointerThe normalized desktop pointer-pad state used by the controller.
signals.audioThe latest analyzed audio values such as level, bass, mid, and treble.
session.cadenceMsThe live-stream update interval used when the desktop panel is streaming continuously.
