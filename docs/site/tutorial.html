Step 1: check prerequisites

Run the prerequisite report first. This does not build anything. It only tells you whether CMake, Python, Git, a compiler, and optional documentation tools are present. 

.\scripts\report-prerequisites.ps1

Step 2: build the project

The build script configures the project with CMake and then compiles it. By default it produces a Debug build. You can also choose Release. 

.\scripts\build-halcyn-app.ps1
.\scripts\build-halcyn-app.ps1 -Configuration Release

Step 3: run the app

The run script builds the app if necessary, starts the HTTP API, and opens the render window. You can choose a startup sample or load a scene file immediately. 

.\scripts\launch-halcyn-app.ps1
.\scripts\launch-halcyn-app.ps1 -Sample 3d
.\scripts\launch-halcyn-app.ps1 -SceneFile .\examples\scene_2d_lines.json
.\scripts\launch-halcyn-app.ps1 -Port 0

If you use -Port 0, Windows chooses any free TCP port. Halcyn then prints the real port in its startup summary so you can connect to the correct address.

Step 4: post JSON into the running app

Open a second PowerShell terminal and post a sample scene. When the request succeeds, the renderer picks up the new scene on the next frame. 

.\scripts\post-example-2d-scene.ps1
.\scripts\post-example-3d-scene.ps1
.\scripts\post-scene-json-file.ps1 -SceneFile .\examples\scene_3d_wireframe_lines.json

What happened behind the scenes?

The API thread

Received your HTTP request, parsed the JSON, validated it, and stored the latest good scene.

The scene store

Created a new immutable snapshot with a higher version number so the renderer could notice the change.

The render thread

Detected the new version, uploaded the scene to GPU buffers, and kept drawing at the target frame rate.

Step 5: use the browser Control Center

If you prefer a web-based workflow, launch the Control Center and open it in your browser. From there you can run bootstrap, build, test, docs generation, smoke checks, and API requests without remembering the individual terminal commands. 

.\scripts\launch-browser-control-center.ps1

Step 6: use the native desktop control panel when you need hardware-aware control

The desktop control panel is the best fit when you want one operator surface beside the renderer window, especially if you need local audio-device selection, instant 2D and 3D switching, or a desktop pointer pad instead of browser permissions. 

.\scripts\launch-desktop-render-control-panel.ps1

Inside that app you can preview JSON, validate the current scene, apply it once, or keep a live stream running while you manipulate the controls.
