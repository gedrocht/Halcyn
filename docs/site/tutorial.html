Step 1: check prerequisites

.\scripts\report-prerequisites.ps1

This tells you whether Python, CMake, Doxygen, audio packages, Visual Studio, and the other helpers needed by the repo are currently available on your machine.

Step 2: open the easiest full setup

.\scripts\launch-visualizer-workbench.ps1

That opens the Visualizer, the browser Control Center, and the unified Visualizer Studio desktop app in separate windows.

Step 3: learn the four main roles

  • Visualizer: draws the current scene.
  • Visualizer Studio: creates scenes from sources and sends them to the Visualizer.
  • Control Center: launches things, runs jobs, and gives you API and docs access.
  • Activity Monitor: shows the shared log stream from the running apps.

Step 4: try the preset scene family

  1. In Visualizer Studio, leave the scene family on Preset scene.
  2. Choose a source mode such as plain text or pointer pad.
  3. Use Preview to inspect the generated scene.
  4. Use Validate and then Apply.

Step 5: try the bar-wall scene family

  1. Switch the scene family to Bar wall.
  2. Try a JSON document, plain text, random values, or audio.
  3. Adjust grid size, shader style, anti-aliasing, and range mode.
  4. Open the preview JSON window so you can study the generated 3D scene.

Step 6: watch the shared logs

.\scripts\launch-activity-monitor.ps1

This opens the sortable, filterable browser view over the shared activity journal. Watching this while you use Visualizer Studio makes the whole system much easier to understand.
